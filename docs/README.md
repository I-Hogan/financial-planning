## Structure

- `ARCHITECTURE.md`: System-level overview, key components, and design decisions.
- `how_to/`: Task-focused guides for accomplishing specific goals.
- `concepts/`: Background explanations, terminology, and domain concepts.

See the README in each subfolder for guidance on what belongs there.
